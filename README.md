# Singly Linked List (Односвязный список)

### Список реализованных методов:

| Метод | Сложность | Описание |
| :--- | :---: | :--- |
| `Head` | $O(1)$ | Возвращает ссылку на первый узел (начало списка). |
| `Tail` | $O(1)$ | Возвращает ссылку на конец (конец списка). |
| `Count` | $O(1)$ | Возвращает количество элементов у списка. |
| `IsReadOnly` | $O(1)$ | Возвращает разрешение можно ли изменять список или нет. |
| `GetEnumerator` | $O(n)$ | Реализует интерфейс IEnumerable<T> для перебора элементов. |
| `Indexer` | $O(n)$ | Доступ к элементам списка (чтение и изменение). |
| `FindElement` | $O(n)$ | Возвращает элемент по индексу. |
| `Clear` | $O(1)$ | Удаляет весь список путём удаления первого узла. |
| `RemoveFirst` | $O(1)$ | Удаляет первый узел. |
| `RemoveLast` | $O(n)$ | Удаляет последний узел. |
| `Insert` | $O(n)$ | Вставляет элемент в список по заданному индексу. |
| `Add` | $O(1)$ | Добавление элемента в конец списка (Добавление происходит за $O(1)$ благодаря Tail). |
| `RemoveAt` | $O(n)$ | Удаление элемента по индексу. |
| `AddFirstElement` | $O(1)$ | Добавление элемента в начало списка. |
| `ShowList` | $O(n)$ | Вывести список элементов. |
| `CopyTo` | $O(n)$ | Скопировать все элементы из списка в полученный массив. |
| `IndexOf` | $O(n)$ | Возвращает индекс существующего элемента, в противном случае возвращает -1. |
| `Contains` | $O(n)$ | Возвращает истину, если элемент есть в списке, в противном случае - ложь. |
| `Remove` | $O(n)$ | Возвращает истину, если получилось удалить нужный элемент, в противном случае - ложь. |

# Binary Search Tree (Бинарное дерево поиска)

[23.03.2026] Тесты BST пока не проведены, но список, реализованных методов пополняется

### Список реализованных методов:

| Метод | Сложность | Описание |
| :--- | :---: | :--- |
| `Head` | $O(1)$ | Возвращает ссылку на корень (начало дерева). |
| `Count` | $O(1)$ | Возвращает количество элементов у дерева. |
| `AddNode` | $O(log n)$ | Добавляет элемент в дереве. |
| `ShowTree` | $O(n)$ | Визуально показывает структуру дерева (Left->Root->Right). |
| `Contains` | $O(log n)$ | Возвращает истину, если данный узел есть в дереве, иначе вернёт ложь. |
| `GetMin` | $O(log n)$ | Возвращает минимальный элемент. |
| `GetMax` | $O(log n)$ | Возвращает максимальный элемент. |
| `Find` | $O(log n)$ | Возвращает нужный узел дерева, иначе, если такого узла нет, вернёт null. |
| `HeightTree` | $O(n)$ | Возвращает максимальную глубину дерева. |
| `CountLeaf` | $O(n)$ | Возвращает количество листьев. |
| `GetLevelNode` | $O(log n)$ | Возвращает глубину нужного узла, если такого узла нет, возвращает -1. |
| `RemoveNode` | $O(log n)$ | Удаляет узел по заданному значению. |
| `IsBalanced` | $O(nlog n)$ | Проверяет сбалансированное ли дерево, или нет (Если у одного из узлов дерева его потомки имеют разность глубин больше 1, то оно несбалансированное). |
| `InOrder` | $O(n)$ | Позволяет пройти дерево в порядке возрастания элементов (Left->Root->Right). |
| `PostOrder` | $O(n)$ | Позволяет пройти дерево снизу вверх: сначала потомков, затем родителя (Left->Right->Root). |
