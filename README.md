| `Tail` | $O(1)$ | Возвращает ссылку на конец (конец списка). |
| `Count` | $O(1)$ | Возвращает количество элементов у списка. |

# 🌿 Binary Search Tree (Бинарное дерево поиска)

### Список реализованных методов:

| Метод | Сложность | Описание |
| :--- | :---: | :--- |
| `Head` | $O(1)$ | Возвращает ссылку на корень (начало дерева). |
| `Count` | $O(1)$ | Возвращает количество элементов у дерева. |
| `Add` | $O(log n)$ | Добавляет элемент в дереве. |
| `ShowTree` | $O(n)$ | Визуально показывает структуру дерева (Left->Root->Right). |
| `Contains` | $O(log n)$ | Возвращает истину, если данный узел есть в дереве, иначе вернёт ложь. |
| `GetMin` | $O(log n)$ | Возвращает минимальный элемент. |
| `GetMax` | $O(log n)$ | Возвращает максимальный элемент. |
| `Find` | $O(log n)$ | Возвращает нужный узел дерева, иначе, если такого узла нет, вернёт null. |
| `HeightTree` | $O(n)$ | Возвращает максимальную глубину дерева. |
| `CountLeaf` | $O(n)$ | Возвращает количество листьев. |
| `GetLevelNode` | $O(log n)$ | Возвращает глубину нужного узла, если такого узла нет, возвращает -1. |
| `Remove` | $O(log n)$ | Удаляет узел по заданному значению. |
| `IsBalanced` | $O(nlog n)$ | Проверяет сбалансированное ли дерево, или нет (Если у одного из узлов дерева его потомки имеют разность глубин больше 1, то оно несбалансированное). |
| `InOrder` | $O(n)$ | Позволяет пройти дерево в порядке возрастания элементов (Left->Root->Right). |
| `PostOrder` | $O(n)$ | Позволяет пройти дерево снизу вверх: сначала потомков, затем родителя (Left->Right->Root). |
| `PreOrder` | $O(n)$ | Позволяет пройти дерево сверху вниз: сначала родитель, затем потомки (Root->Left->Right). |
| `LevelOrder` | $O(n)$ | Позволяет пройти дерево сверху вниз: сначала соседние узлы, затем все соседи соседей. |
| `IsReadOnly` | $O(1)$ | Позволяет узнать: можно ли изменять данную структуру или нет. |
| `CopyTo` | $O(n)$ | Позволяет скопировать часть дерева в массив. |

# 🫧 Сортировка пузырьком (O(n), O(n²), O(n²))

### Описание работы пузырьковой сортировки:

Сортировка пузырьком - это зачастую первый алгоритм, с которым знакомятся новички при изучении основ программирования. Данная сортировка считается одной из самых "тяжелых", так как для каждого из n элементов массива приходится совершать полный проход по всем оставшимся n-1 элементам. Вследствие этого общая сложность алгоритма составляет $O(n^2)$.

Реализовать данную сортировку полезно для саморазвития или в рамках подготовки к поиску первой работы в ИТ. Однако в промышленном программировании её практически не используют из-за низкой эффективности на больших объёмах данных.

Математическое обоснование производительности:
* Для 100.000 элементов выполнение алгоритма занимает примерно 100 секунд ($100.000^2$ / $10^8$).
* Для 1.000.000 элементов потребуется уже около 10.000 секунд, что составляет примерно 2.7 часа.

Технический момент: внутри цикла вводится булева переменная. Если за весь проход по элементам не было ни одного обмена (свапа), это означает, что массив уже отсортирован.

Для сравнения: современные оптимизированные алгоритмы справляются с миллионом элементов менее чем за одну секунду.

# ⚡ Сортировка выбором (O(n²), O(n²), O(n²))

### Описание работы сортировки выбором:

Сортировка выбором — это более эффективная версия по сравнению с пузырьковой сортировкой. Её преимущество в том, что элементы не меняются местами на каждой итерации. Алгоритм действует более осмысленно: вы проходите по всем элементам внешним циклом, а во внутреннем цикле (начиная с i+1 и до конца массива) ищете подходящий элемент.

В процессе прохода вы сохраняете индекс минимального (или максимального) значения в зависимости от логики, переданной в Func. Только после завершения внутреннего цикла выполняется единственный обмен (свап), при котором найденный элемент ставится на свою итоговую позицию. Это значительно сокращает количество операций записи в память.

Технический момент: Сортировка выбором - это единственный алгоритм из простых, который не имеет условия досрочного выхода, так как на каждом шаге он обязан гарантированно найти минимум в неотсортированной части, просмотрев ее до конца.

# 🧩 Сортировка вставками (O(n), O(n²), O(n²))

### Описание работы сортировки вставками:

Сортировка вставками - это третья сортировка, которая тоже лучше, чем пузырьком, хотя и имеет сложность в общем O(n²). Смысл алгоритма в том, что мы берём каждый элемент (начиная со второго) и тянем его назад к началу массива. Во внутреннем цикле мы сравниваем его с левыми соседями и, если условие выполняется, сдвигаем их вправо. Как только нашли подходящее место или дошли до нуля - вставляем элемент туда.

# ⚡🚀🔥 Быстрая сортировка (O(nlogn), O(nlogn), O(n²))

### Описание работы Быстрой сортировки:

Быстрая сортировка использует стратегию "разделяй и властвуй". Рекурсивная реализация проста, но на больших массивах может переполнить стек. Для больших данных лучше использовать итеративную реализацию с собственным стеком в куче. Алгоритм: выбираем pivot, делим массив на две части, повторяем для каждой части, пока размер подмассива не станет 1.

# 🔴 Max-Heap  (Максимальная бинарная куча)

# Список реализованных методов:

| Метод | Сложность (средняя) | Описание |
| :--- | :---: | :--- |
| `IsEmpty` | `O(1)` | Возвращает ложь, если куча непустая, иначе - истину. |
| `IsReadOnly` | `O(1)` | Возвращает ложь, так как структура изменяемая. |
| `Count` | `O(1)` | Возвращает количество элементов. |
| `Add` | `O(log n)` | Вставка элемента в кучу. |
| `PrintAsTree` | `O(n)` | Вывести рекурсивно дерево. |
| `Pop` | `O(logn)` | Удалить максимальный элемент из кучи. |
| `Peek` | `O(1)` | Взять максимальный элемент. |
| `Clear` | `O(1)` | Очистить кучу. |
| `Contains` | `O(n)` | Проверяет, есть ли такой элемент. |
| `GetEnumerator` | `O(n)` | Возвращает итератор для перебора элементов. |
| `CopyTo` | `O(n)` | Копирует все элементы в массив. |
| `Remove` | `O(n)` | Не поддерживается данной структурой📌. |

📌 Примечание: обычно метод Remove предполагает удаление любого элемента в контейнере, если он есть. Но проблема в том, чтобы удалить нужный элемент придется пройтись по массиву, чтобы найти этот элементов, но тогда сложность будет O(n), что не подходит для бинарной кучи, так как она предполагает работу за O(logn) ради чего она и была создана.
